KQL Playground

Browser-based KQL query engine. Type a query against pre-loaded sample tables and run it — no backend, no signup, no Sentinel tenant required. Same translation logic as my Invoke-KqlPS PowerShell module, ported to run in your browser.

Preview build. The UI is wired up but the KQL-to-SQL engine is still being ported from PowerShell to JavaScript. Clicking Run on any query right now will show a placeholder result. Full functionality landing in a follow-up release. Watch the blog for the announcement.

How this will work

When the engine ships, this page will load three things from CDN:

  • sql.js — SQLite compiled to WebAssembly. ~600 KB. Initializes an in-memory database in your browser.
  • Sample data — Sanitized CSV exports of common tables (DeviceLogonEvents, etc.) bulk-loaded into the WASM SQLite at page load.
  • kql-translator.js — Port of the PowerShell Invoke-KqlPS translator: parses KQL, emits SQL, runs against the WASM database, formats results.

Nothing leaves your browser. No backend. No analytics. The query bar is just a <textarea> talking to a 600 KB WASM module — which means it works offline once the page is cached.

Why a portfolio site has a query playground

Most security portfolios show screenshots of dashboards. That proves nothing — anyone can take a screenshot. A working playground means you can verify the engine actually translates KQL to SQL correctly by running queries yourself. It's the hardest signal to fake.
