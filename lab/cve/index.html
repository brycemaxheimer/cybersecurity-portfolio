CVE / KEV / EPSS Browser

Coming soon.

A snapshot-backed browser for three public vulnerability feeds:

  • NVD CVE feed — the canonical CVE catalog with CVSS scoring, affected configurations, and references.
  • CISA KEV — Known Exploited Vulnerabilities. The "actually being used in the wild" subset of CVE.
  • FIRST EPSS — Exploit Prediction Scoring System. Probability that a CVE will be exploited in the next 30 days.

Refresh model

The feeds are pulled by a GitHub Action on a nightly schedule, slimmed to the fields the browser needs, committed back to the repo, and redeployed by Cloudflare Pages on the next push. The Action's source will live at .github/workflows/refresh-cve-feeds.yml once built. No client-side API calls; no rate-limit risk.

Planned features

  • Filter by severity (low/medium/high/critical), KEV-listed, EPSS > threshold.
  • Sort by published date / EPSS score / CVSS / KEV-add date.
  • Click into a CVE for full description, CVSS vector, references.
  • Cross-link to the MITRE ATT&CK Explorer when the CVE is mapped to a technique.
  • "Last refreshed" timestamp visible at the top of the page.

Status: not yet built. Underlying source: SOC Dashboard / Modules / Update-Cve*.ps1.
